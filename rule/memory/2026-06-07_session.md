## 完成事项 (8大功能 + 版本升级)

### 1. Extended Thinking / Reasoning
- 修复 AnthropicProvider 流式 thinking bug：`_cur_thinking` 从未被设置，所有 thinking 内容静默丢弃 → 改用 `cur_thinking_text` 局部变量
- 修复 OpenAI/DeepSeek 流式 usage 丢失：加 `stream_options={'include_usage': True}`
- `/thinking` 命令 toggle，config: `thinking.enabled + budget_tokens`
- Anthropic: `thinking={"type":"enabled","budget_tokens":N}`；DeepSeek: `extra_body`

### 2. Prompt Caching
- Anthropic: system prompt 包装为 `[{type:text, cache_control:{type:ephemeral}}]`，最后一个 tool 也加
- `_is_cache_error()` 自动降级——API 不支持 cache_control 时回退无缓存重试
- TokenUsage 新增 `cache_read/cache_write` 字段
- DeepSeek: 服务端自动缓存，无需客户端注解

### 3. Structured Output / JSON Mode
- `response_format` + `tool_choice` 全链路透传: config→LLMClient→provider→API
- OpenAI: `response_format={"type":"json_object"}` + `tool_choice`
- 默认不启用，按需使用

### 4. Streaming Tool Results
- ReadTool: 大文件(>100行)逐行流式，每10行节流
- WebFetchTool: 流式进度 "Fetching url... → 200 text/html | 15234 chars → Extracting..."
- WebSearchTool: "Searching: query... → Found 8 results"
- BrowserFetchTool: "Launching browser... → Loading url... → Page loaded → Browser closed"
- agent 并行路径修复：lambda 闭包传工具名，CLI 显示 `[dim]ToolName[/dim] │`
- CLI `on_tool_output` 增加 `tool_name` 参数

### 5. Worktree Workspace Isolation
- 新建 `tools/worktree.py` (~290行)：WorkdirRef + WorktreeManager + EnterWorktreeTool + ExitWorktreeTool
- `/worktree create/enter/exit/list` 命令
- 进入 worktree 时所有文件工具重注册指向新路径
- 系统提示词标注 `ISOLATED WORKTREE`
- 未提交更改保护（exit --remove 拒绝）
- 路径: `.tudou_agent/worktrees`，repo内自动切 `/tmp/tudou_agent_worktrees`

### 6. Skills GitHub 安装
- 升级 `/skills` 为子命令系统
- `/skills install owner/repo` — git clone → 自动找 SKILL.md → 复制到 builtin_skills/ → 注册
- `/skills search <keyword>` — 按名称/描述/标签搜索
- URL 正则支持: `owner/repo` / `github.com/owner/repo` / `https://github.com/owner/repo.git`
- `_find_git_exe()` — Windows 自动探测 4 个常见 git 安装路径 + PATH

### 7. Windows 低完整性进程沙箱
- 新建 `tools/sandbox.py`：SandboxConfig + run_sandboxed + EnableSandboxTool
- 核心 API: OpenProcessToken → DuplicateTokenEx → SetTokenInformation(TokenIntegrityLevel=Low S-1-16-4096) → CreateProcessWithTokenW
- `/sandbox on/off/status` 人控，EnableSandboxTool 单向(LLM只能开不能关)
- 非 Windows 自动退到普通 subprocess

### 8. 断点恢复 (Checkpoint)
- SessionStore 新增 checkpoints 表：save_checkpoint / load_checkpoint / delete_checkpoints
- agent 每轮工具执行后 + 最终回复前自动存 checkpoint（每对话最多3个）
- 对话成功完成自动清理
- `/resume` 裸敲 → `_find_orphaned_checkpoint()` 自动找最近崩溃会话并恢复
- `/resume <id>` 自动检测并优先加载 checkpoint 而非普通 messages

### 其他修复
- `_show_help` 重写：f-string 跨行语法错误 → 改用 list + join
- help 表格内嵌 `|` 改 `,` 解决 markdown 解析断裂
- `SLASH_COMMANDS` 补全列表新增 `/thinking /sandbox /worktree /subagent /tasks`
- `SUBCOMMANDS` 新增 sandbox/worktree/skills/resume 子命令补全
- config.yaml 两次恢复（打包目录被清洗）
- `import tempfile` 缺失修复
- `_is_cache_error` 关键词收窄（去掉过于宽泛的 `cache` 和 `invalid_request_error`）
