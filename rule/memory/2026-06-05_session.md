## 完成事项

### 1. 任务面板动态波浪 + 清除
- 经过多轮迭代，最终方案：纯 `sys.stdout.write` + ANSI 码，和 spinner 同通道
- **波浪渐变**：`_wave_chars(text, t, extra_sgr)` — 逐字 ANSI true-color `\x1b[38;2;0;g;bm`，可选叠加状态样式（`\x1b[9m` 删除线等）
- **面板外壳**：`╭─╮│╰─╯` 字符搭建，全部波浪渲染
- **动态刷新**：`_panel_animate()` 线程每 100ms 用新时间戳重绘
- **清除旧面板**：`\x1b[{N}A` + `\x1b[J` 移回开头清到尾（不用 `\x1b7`/`\x1b8`，避免和 Rich 冲突；不用 `\x1b[M]`，兼容性问题）
- **与 spinner 互斥**：有任务→面板动画，无任务→spinner，永不共存
- **生命周期**：`_maybe_show_thinking()` 统一入口；`on_pre_tool`/`on_stream_token`/异常/结束时停面板动画

### 2. 审批样式重设计
- 新增 `render_approval_block(name, arguments)` 在 `renderer.py`
  - `─` 分隔线（全宽 dim）
  - 工具标签淡蓝色（如 "Create file", "Run command"）
  - 详情行：`file:` / `commands:` / `pattern:` 等前缀 + 淡蓝色内容
  - Write 工具：`╌` 分隔线 + 代码预览（语法高亮 + 行号）+ `╌` 分隔线
  - Edit 工具：`╌` 分隔线 + unified diff + `╌` 分隔线
  - 返回问题文本供 `prompt_choice` 使用

### 3. 代码语法高亮
- `renderer.py` 新增 `_LANG_MAP`（扩展名→语言映射）
- 批准块中 Write 代码预览使用 `Syntax(code, lang, theme='native', line_numbers=True)`

### 4. 三选项审批 + auto-approve
- `prompt_choice` 改为返回 `int`（0/1/2/-1），支持任意数量选项
- 选项：Yes / "Yes, allow all edits during this session (shift+tab)" / No
- `shift+tab` 直达选项2
- 选选项2 设 `_auto_approve = True`，本轮对话后续工具不再询问
- `_auto_approve` 每轮对话开始时重置

### 5. 颜色统一
- 工具名（`render_tool_call` 的 `● Edit(...)` 等）：`[bold rgb(135,206,250)]` 淡蓝
- 审批块标题（"Create file"等）：`[bold rgb(135,206,250)]` 淡蓝
- 详情行内容：`[rgb(135,206,250)]` 淡蓝

## 关键经验
- **不要混用 Rich 和原始 ANSI 码**：两套系统同时写 stdout 会穿插乱码。面板和 spinner 只能二选一运行。
- **`\x1b7`/`\x1b8` 可能被 Rich 内部占用**：改用 `\x1b[{N}A` + `\x1b[J]` 更可靠。
- **`\x1b[M]`（Delete Line）终端兼容性差**：用 `\x1b[J`（Clear to End）替代。
- **Rich Live 和 spinner 线程不能共存**：Live.stop() 和 spinner 的 sys.stdout.write 会穿插。
