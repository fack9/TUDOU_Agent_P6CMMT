## 会话概要

对 TUDOU_agent v2.12.1.3 进行了 6 项功能优化 + 3 个 bug 修复。

## 功能优化

### 1. Skill 自动调用系统
- **SkillTool**: LLM 可自主调用技能，加载技能正文并按其指令执行
- **SkillListTool**: 分页列出所有可用技能（50/页），供 LLM 按需发现
- **SkillSearchTool**: 按名称/描述/标签搜索技能，评分排序（名称匹配+10，描述+5，标签+3）
- 与 `/activate` 共存：前者用户手动预设，后者 LLM 动态匹配

### 2. 技能列表按需加载（Token 优化）
- 从 system prompt 移除 150+ 技能的完整列表（之前每轮浪费 ~2000 tokens）
- LLM 通过 SkillList/SkillSearch 按需查询
- 涉及清理：`build_system_prompt`、`build_messages`、`run_conversation` 的 `skills_summary` 参数全部移除
- `_get_skills_summary` 方法删除

### 3. Bash 错误输出包含 stderr
- `stderr=subprocess.STDOUT` 改为独立 PIPE
- 命令失败时 error 字段包含 `Exit code + [stderr]` 内容
- stderr 行也通过 `on_output(line, 'stderr')` 流式输出（品红色）
- 成功时 stderr 附加到 output 末尾供参考

### 4. 工具参数校验兜底
- `registry.execute` 异常时显示 `TypeError: ... [Called: ToolName({args})]`，含调用上下文
- `Glob` 新增空 pattern 防御："Pattern is required and cannot be empty. Example: **/*.py"

### 5. 流式输出多余空行修复
- 根因：`render_text` 每行前后各加一个空行，流式输出每行都触发 → 3行变9行
- 修复：新增 `render_tool_output_line` 紧凑方法，`on_tool_output` 改用此方法
- `render_text` 去掉首尾空行包装
- `render_tool_result` 去掉末尾空行
- `render_tool_call` 去掉首行空行

### 6. 工具输出颜色区分
- 浅蓝色 rgb(135,206,250)：stdout、成功结果 ✓
- 品红色 rgb(255,100,180)：stderr、错误结果 ✗
- `on_output` 回调签名扩展：`(line, stream='stdout')`，向后兼容旧式单参数调用
- Agent lambda：`lambda line, stream='stdout': on_tool_output(line, n, stream)`

## Bug 修复

1. **SYSTEM_PROMPT 语法错误**：Edit 工具 JSON 参数将 `\n` 解释为物理换行符，导致 Python 字符串未闭合 → 用 `\\n` 重写修复
2. **Memory→Paths 引用方向错误**：静态提示词写 "see Paths section ABOVE"，但 `build_system_prompt` 把 Paths 追加在末尾 → 改为 "shown in the Paths section"
3. **Skill 初始化顺序**：`_register_tools` 在 `self.skills` 创建之前调用 → 移动 skills 初始化到 `_register_tools` 之前
4. **`_remove_skill` 重建 registry**：创建新 SkillRegistry 导致 SkillTool 引用失效 → 改用 `invalidate_cache` + `discover` 原地更新

## 涉及文件

- `tools/skill_invoke.py` — 新增 SkillListTool, SkillSearchTool
- `tools/bash.py` — stderr 分离 + 流式
- `tools/registry.py` — 参数校验错误增强
- `tools/glob_tool.py` — 空 pattern 防御
- `context/manager.py` — 移除 skills_summary, 修复 Memory 引用, 修复语法错误
- `agent.py` — 移除 skills_summary, lambda 支持 stream 参数
- `cli.py` — 注册 3 个 skill 工具, 移除 _get_skills_summary, on_tool_output 支持 stream
- `ui/renderer.py` — render_tool_output_line 紧凑方法 + 颜色, render_text/render_tool_result/render_tool_call 去空行
