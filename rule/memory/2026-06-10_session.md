## 会话概要

对 TUDOU_agent v2.12.1.3 → v2.13.5.7 进行了大量功能优化和 bug 修复。

## 功能优化

### 1. Skill 自动调用系统
- **SkillTool**: LLM 可自主调用技能，加载技能正文并按指令执行
- **SkillListTool**: 分页列出所有可用技能（50/页），供 LLM 按需发现
- **SkillSearchTool**: 按名称/描述/标签搜索，评分排序（name+10, desc+5, tags+3）
- 与 `/activate` 共存：用户手动预设 vs LLM 动态匹配

### 2. 技能列表按需加载（Token 优化）
- 从 system prompt 移除 150+ 技能完整列表（每轮省 ~2000 tokens）
- LLM 通过 SkillList/SkillSearch 按需查询
- 清理：`build_system_prompt`、`build_messages`、`run_conversation` 的 `skills_summary` 参数全部移除
- `_get_skills_summary` 方法删除

### 3. Bash 错误输出包含 stderr
- `stderr=subprocess.STDOUT` 改为独立 PIPE
- 命令失败时 error 字段包含完整 stderr
- stderr 行通过 `on_output(line, 'stderr')` 流式输出
- 成功时 stderr 附加到 output 末尾

### 4. 工具参数校验兜底
- `registry.execute` 异常时显示 `[Called: ToolName({args})]` 调用上下文
- `Glob` 新增空 pattern 防御

### 5. 流式输出多余空行修复
- 根因：`render_text` 每行前后各加空行，流式输出 3 行变 9 行
- 修复：新增 `render_tool_output_line` 紧凑方法
- `render_text`、`render_tool_result`、`render_tool_call` 去掉冗余空行

### 6. 工具输出颜色区分
- 浅蓝 rgb(135,206,250)：stdout、成功结果
- 品红 rgb(255,100,180)：stderr、错误结果
- `on_output` 回调签名扩展：`(line, stream='stdout')`，向后兼容
- Agent lambda：`lambda line, stream='stdout': on_tool_output(line, n, stream)`

### 7. 工具输出阶段颜色
- `render_pre_tool`：黄色 → 白色（执行中状态）
- `render_tool_call`：新增 `success` 参数，成功浅蓝/失败品红
- 执行中白色 → 完成后着色判定

### 8. MEMORY.md 种子 + 自动加载
- 创建 `memory/MEMORY.md` 种子文件
- `ContextManager` → `MemoryManager.build_context_block()` 加载管线已完整
- SYSTEM_PROMPT 修复：`Read them` → `Do NOT use the Read tool to re-read them`
- 新增反过度探索指引

### 9. SYSTEM_PROMPT Bug 修复
- Edit 工具 JSON 参数将 `\n` 解释为物理换行 → 语法错误 → 用 `\\n` 修复
- Memory→Paths 引用方向错误：`above` → `shown in the Paths section`

### 10. Taste Skill 集成
- 从 GitHub API 下载 13 个 taste-skill 到 `builtin_skills/taste-*/`
- 总计 163 个技能（150 + 13）

### 11. Explore 子代理工具
- `ExploreTool`：接收 description + prompt，创建只读子代理
- 子代理并行运行 Read/Glob/Grep/Bash/WebSearch/WebFetch
- UI：`render_explore_start` + `render_explore_summary`
- 流式报告子工具调用（explore-pre 流类型）

### 12. /context 输出重写
- 20 块视觉条形图（灰色 system + 紫色 messages + 灰色 free）
- 分类明细（sys prompt / tools / memory / skills / messages / free / buffer）
- Memory files / Skills / Suggestions 三个段落
- 格式：`⎿` 前缀 + 缩进

### 13. AskUserQuestion 表单升级
- 从 `input()` 文本循环 → prompt_toolkit 多标签表单
- 标签栏导航（← → 切换标签，✔ Submit）
- 多选复选框（Space 勾选）
- ANSI 修复：改用 prompt_toolkit 格式化文本元组
- 取消处理：返回 `__cancelled__` → LLM 收到拒绝提示

### 14. 模型上下文限制检测
- `MODEL_TOKEN_LIMITS` 扩展至 42 个模型（含 DeepSeek v4 1M、Claude 200K、Gemini 1M 等）
- 三级检测：用户配置 → API 查询 (/v1/models, 带缓存) → 硬编码表
- `/context` 显示实际模型上下文限制 + 配置覆盖警告

## Bug 修复

1. SYSTEM_PROMPT 语法错误（Edit 工具 \n 转物理换行）
2. Memory→Paths 引用方向错误
3. Skill 初始化顺序（_register_tools 在 skills 创建之前）
4. `_remove_skill` 重建 registry 导致 SkillTool 引用失效
5. Edit 工具 `</parameter>` 注入
6. prompt_form 的 `fg:ansigreen` 样式键错误
