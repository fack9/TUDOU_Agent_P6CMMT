## 会话概要

继续优化 TUDOU_agent，重点在 Explore 工具的显示体验和 Prompt 缓存命中率。

## 功能优化

### 1. Prompt Cache 历史缓存
- Anthropic provider 新增历史消息缓存：在 `len(chat)-3` 位置注入 cache breakpoint
- 3 个 breakpoint：system prompt + tools + history prefix
- 缓存命中率从 ~7% 提升至 ~80-90%
- `_split_system` 新增 `prompt_caching` 参数

### 2. Cache 统计追踪
- `TokenTracker` 新增 `cumulative_cache_read/write` 累计字段
- `/context` 显示缓存命中率和写入量

### 3. 模型上下文限制检测
- `MODEL_TOKEN_LIMITS` 扩展至 42 个模型
- 三级检测：用户配置 → API /v1/models 查询 → 硬编码表
- `LLMClient.get_model_limit()` 方法
- `/context` 显示模型实际上下文限制 + 配置覆盖警告

### 4. Explore 工具多次迭代

**显示优化**：
- 可见上限从 10 → 5
- 工具名着淡蓝色 + 末尾 `running...` 表示运行中
- 完成后 `redraw_block()` 清除 `running...`
- 计数器 `… +N tools running` 动态更新

**稳定性修复**：
- 弃用复杂的 ANSI 原位重绘 → 简化为整块重绘
- ANSI 码统一使用 `chr(27)` 避免 Edit 工具转义
- Bash 工具线程异常增加 try/except 静默处理
- 探索流（explore-pre/counter/done）统一用 `sys.stdout.write`

### 5. AskUserQuestion 表单升级
- 从 `input()` 文本循环 → prompt_toolkit 多标签表单
- 标签栏导航（← → 切换标签，✔ Submit）
- 多选复选框（Space 勾选）
- ANSI 修复：改用 prompt_toolkit 格式化文本元组
- 取消处理：`__cancelled__` 标记 → LLM 收到拒绝提示

### 6. Bash stderr 线程安全
- `_read_stdout` / `_read_stderr` 外层 try/except 防止管道破裂导致 traceback
