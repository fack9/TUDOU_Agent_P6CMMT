## 完成事项 (9大功能 + 1个版本升级)

### 0. 版本号升级
- `utils/constants.py`: `VERSION = '1.17.6.7'`
- `version_info.txt`: 全部 `1.16.4.5` → `1.17.6.7` (含 filevers/prodvers 元组)

### 1. Web 工具优化 (4项)
**新建 `tools/web_utils.py`**:
- TTLCache: dict+时间戳 TTL 缓存（15分钟），`_fetch_cache` / `_search_cache` 两个单例
- retry_request(): 429/502/503/504 + ConnectError/ReadTimeout/RemoteProtocolError 3次指数退避
- html_to_markdown(): BS4实现HTML→Markdown，单函数递归 `_node_to_md()`
- check_robots_txt(): 域名级1小时缓存，fail-open，路径前缀匹配

**修改 `tools/web_search.py`**: cache + retry (key=`q:{query}`)
**修改 `tools/web_fetch.py`**: cache (key=`u:{url}`) + retry + markdown + robots.txt + AssistLLM 提炼
- extract 参数默认 false，prompt 不触发提炼
- extract=true + 已配 extraction → 调副模型 (独立 API key/base_url 支持)
- 配置: `tools.web_fetch.extraction_model/api_key/base_url`

### 2. BrowserFetch (Playwright)
**新建 `tools/browser_fetch.py`**: headless Chromium 渲染 SPA 页面
- 复用 `html_to_markdown()` + `_fetch_cache` (key=`b:{url}`)
- wait_selector 参数 + try/finally 浏览器关闭
- 依赖: `playwright>=1.40.0` + Chromium 148.0.7778.96

### 3. 内容感知压缩
**新建 `context/content_compressor.py`** (~350行):
- **ContentDetector**: 检测 JSON/Code/Log/HTML/Text 5种类型，纯启发式零依赖
- **SmartCrusher**: JSON 同构数组压缩比 99%+ (6万→5百字符)
- _compress_code(): 保留 import/函数签名/注释，函体采样
- _compress_log(): ERROR全保留，WARNING前20条，INFO去重计数
- _compress_text(): 保留首尾段，中间均匀采样
- compress_content() + content_snippet() 路由函数
- 接入: `TokenTracker.truncate_tool_result()` + `ContextCompressor._content_snippet()`

### 4. CCR 可逆压缩
**新建 `context/ccr.py`**: CCRStore — OrderedDict + UUID key，LRU淘汰 (max 500)
**新建 `tools/ccr_retrieve.py`**: RetrieveTool — 按 rid 取回原文
- agent.py `_append_tool_messages()`: 截断后追加 `[CCR:abc123 | original: N chars]`
- 小输出不触发CCR，LLM按需惰性检索

### 5. 并行工具执行
- agent.py 循环体重构为三阶段: 验证分类 → 只读并行 → 写顺序
- 只并行 `is_read_only=True` 的工具，ThreadPoolExecutor(max_workers=min(n,8))
- 写工具(Bash/Write/Edit)始终顺序
- worker线程只跑 `reg.execute()`，消息append在主线程

### 6. Bash 流式输出
- `tools/bash.py`: `subprocess.run` → `Popen` + 独立线程逐行读取 + `on_output` 回调
- stdout+stderr 合并 (`stderr=STDOUT`)
- `ToolRegistry.execute()` 透传 `on_output`，不支持的工具 TypeError 降级
- CLI 实时渲染: `  │ {line}`

### 7. LLM 调用重试
- `llm/client.py` `complete()`: 检测类名+消息+HTTP状态码，3次指数退避
- 可重试: RateLimit/Timeout/Connection/InternalServer/5xx/429
- 不可重试: 4xx (非429)
- 配置: `max_llm_retries: 3`

### 8. 子 Agent 委派
**新建 `tools/agent_delegate.py`**: Agent 工具
- 创建独立 TUDOU_Agent 实例（共享 LLM，独立 context/CCR）
- 默认只读工具集，tools 参数可指定
- 递归保护：子 Agent 移除 Agent 工具
- 结果 metadata 包含 tools_used/tokens/duration

### 9. Hook 系统
**新建 `hooks/` 模块** (3文件):
- HookEvent 枚举: on_agent_start/stop, on_tool_before/after, on_compress
- HookManager: YAML加载、模板替换(`{name}`)、阻塞/非阻塞调度
- on_tool_before 阻塞型 → 返回非零阻断工具
- 非阻塞事件后台线程执行，独立超时保护
- 配置路径: `~/.tudou_agent/hooks.yaml` + 项目 `.tudou_agent/hooks.yaml`
- 接入: agent.py 全部 5 个事件点 + cli.py 加载
