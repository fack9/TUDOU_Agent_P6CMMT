## Plan Mode 提问修复

### 问题
LLM 在 plan mode 中写计划书时可能向用户提问，但用户界面只有"批准/修改/拒绝"三个按钮，无法回答。

### 修复
- `tools/plan_mode.py`: `PLAN_MODE_INSTRUCTIONS` 新增第2条——"Resolve ambiguities FIRST: ASK THE USER DIRECTLY, do NOT write questions into the plan file"
- `tools/ask_user.py`（新建）: `AskUserQuestionTool` — LLM 可调用此工具在 plan mode 中向用户提问
- `ui/input_handler.py`: `prompt_user_response()` — 终端收集用户回答
- `cli.py`: 注册工具 + `get_supplements` 回调，将用户回答注入到对话中

### 流程
```
LLM进入PlanMode → 读到指令 → 调AskUserQuestion → CLI渲染问题 → input()收集答案
→ supplements注入 → LLM看到回答 → 写确定性计划 → ExitPlanMode
```

## 面板残留修复

### 根因
1. **线程竞态**: 面板动画线程和主线程同时写 stdout，ANSI转义码交错
2. **散落复位**: 4处 `_last_panel_lines = 0` 在斜杠命令/清除会话时将计数清零，导致面板清不掉
3. **错误方向**: 尝试用 `\x1b7`/`\x1b8` 光标保存/恢复替代行计数，结果保存点太旧，清除时把全部 agent 输出都删了

### 修复（最终版）
- 回退到行计数方案，保留三个核心修复:
  - `_panel_lock`: 面板 stdout 写入加锁
  - 删除 4 处散落的 `_last_panel_lines = 0`
  - `_panel_animate` 持锁后二次检查 `_panel_running`
- 面板始终在 stdout 最底部——主线程写任何输出前先 `_stop_panel_animation()`，行计数始终准确
